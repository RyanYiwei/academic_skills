# 报告框架 — 通用学术论文

本文件供 `skill.md` 在判定论文为非机器人/控制领域的通用学术论文后读取，定义完整的分析报告框架。

---

输出完整分析报告。风格：专业严谨，面向同领域研究生或高年级本科生。全程中文，专业术语保留英文原文并附简短解释。

严格按照以下 8 个部分输出，不得省略。

---

### 0. 摘要翻译

将论文摘要原文翻译为中文，保持学术语言风格，不做删减。

---

### 1. 问题定义与动机

**a) 要解决的问题**
明确说明：
- 这篇论文研究的是什么问题？输入是什么、输出是什么？
- 为什么这个问题难？现有手段的根本瓶颈在哪里？

**b) 现有方法的痛点**
现有主流方法（列出具体方法名）的核心局限性，要具体，不泛泛而谈。

**c) 核心假设与 insight**
作者的核心 insight 是什么？用 2-3 句话概括本文认为"为什么这样做能解决上述痛点"。

---

### 2. 方法精读

> ⚠️ 直接对照论文原文的 Methods 部分，保留论文自身的章节结构。逐节展开，不做合并压缩。

按照论文 Methods 的原始小标题逐节分析，每节格式如下：

---

**`§X.X 原始节标题（英文）`**

**这一节在做什么**：1-2 句话说清楚本节的核心目标。

**技术细节**（按实际涉及的维度展开，没有的维度跳过）：

- **数学公式**
  - 写出本节所有关键公式（使用 LaTeX 或清晰的文字表达）
  - 逐项解释每个符号：含义是什么、取值范围
  - 说明公式的输入从哪里来、输出去哪里用

- **模型 / 网络结构**
  - 整体架构类型：MLP / CNN / Transformer / RNN / GNN / Diffusion / 混合
  - 各子模块的输入维度、隐层维度、输出维度
  - 编码器/解码器设计：如何处理不同模态的输入（文本、图像、图结构等）
  - 是否使用注意力机制：self-attention / cross-attention / 位置编码方式
  - 时序建模：是否有 LSTM / GRU / causal Transformer / sliding window
  - 归一化方式：BatchNorm / LayerNorm / RMSNorm / 无
  - 激活函数选择（GeLU / ReLU / Swish 等）
  - 多模块间权重是否共享

- **训练方式**
  - 任务类型：监督学习 / 自监督 / 对比学习 / 生成模型 / 强化学习 / 混合
  - Loss 函数构成：各项 term 的含义与权重
  - 优化器与学习率调度（warmup、cosine decay 等）
  - Batch size、训练总步数/轮数
  - 多阶段训练（pre-train → fine-tune → RLHF 等）
  - 正则化手段（dropout、weight decay、gradient clipping、label smoothing 等）
  - 数据增强策略（如有）

- **算法 / 推理流程**（如有独立于训练的推理设计）
  - 推理时的具体步骤（如 beam search、chain-of-thought、sampling 策略）
  - 时间/空间复杂度分析

- **改进策略**
  - 相比 baseline，作者做了哪些针对性修改？
  - 每个改动是为了解决什么具体问题（过拟合 / 训练不稳定 / 泛化 / 效率等）

**解决了什么问题**：这个设计对应 Introduction 里提出的哪个具体痛点？

**在整体框架中的作用**：这一节的输出如何流入下一个模块？去掉这一节整个系统会在哪里断掉？

---

> 如果某节内容极少（如纯 notation 定义），可附在上一节末尾，注明"§X.X 符号定义"。

---

### 3. 方法总结（基于精读提炼）

**a) 整体 Pipeline（速览）**
用 4-6 步概括整个方法的数据流：输入 → 各处理步骤 → 输出。比方法精读更精简，侧重模块间的逻辑关系。

**b) 核心设计决策**
从方法精读中提炼出 2-4 个最关键的设计选择，说明"为什么这样选而不是用更简单的方案"。

---

### 4. 与现有方法对比

**a) 本质区别**
与该领域主流方法最根本的不同在哪里？

**b) 核心贡献**（编号列出）

**c) 适用场景与局限**
- 什么任务/数据分布下这个方法更有优势？
- 明确不适用的场景（数据量要求、计算资源、任务类型限制等）？

**d) 对比表格**

| 方法 | 核心思路 | 优点 | 缺点 | 本文改进点 |
|------|----------|------|------|-----------|
| 本文方法 | | | | |
| 对比方法 1 | | | | |
| 对比方法 2 | | | | |

---

### 5. 实验分析

**a) 实验设置**
- 数据集（名称、规模、领域、train/val/test 划分）？
- Baseline 方法有哪些？
- 评估指标（名称 + 含义）？
- 其他重要设置（硬件、随机种子、few-shot/zero-shot 设定等）？

**b) 关键结果**
最具代表性的实验数据和结论，数字要具体。
- 与 baseline 对比的核心提升点
- 是否在多个数据集上一致？

**c) 消融实验**
作者去掉哪些模块后性能下降最显著？这说明哪个设计是最关键的贡献？

**d) 局限性与潜在失效场景**
- 泛化能力如何（domain shift、out-of-distribution 输入）？
- 计算开销：训练时间、推理延迟、内存占用？
- 对数据质量/数量的敏感性？

---

### 6. 实际应用考量

**a) 开源情况**
代码 / 模型权重 / 数据集是否开源？链接？

**b) 复现关键细节**
- 重要超参数（学习率、batch size、模型大小、训练步数等）
- 容易被忽略的训练技巧（数据预处理方式、warmup 设置、特殊初始化等）

**c) 迁移潜力**
- 能否迁移到其他任务或领域？需要改动哪些部分？
- 这篇论文的哪个模块/思路可以作为其他工作的组件复用？

---

### 7. 总结与个人思考

**a) 一句话核心思想**（≤ 20 字）

**b) 速记版 Pipeline**（3-5 步，不用论文术语，直白具体）

示例格式：
1. 输入原始文本，用预训练模型提取上下文表示
2. 用对比学习拉近语义相似样本、推远不相似样本
3. 在目标任务上 fine-tune，输出最终分类结果

**c) 值得关注的开放问题**
这篇论文没有解决、但读完后你觉得值得继续研究的问题（1-3 条）。

---

## 输出规范

- 若论文未涉及某子项，注明"论文未涉及"而非跳过
- 重点在**方法精读**和**实验分析**，引言与结论不过度展开
- 专业术语保留英文原文，首次出现时附中文解释
- 完成后询问："还需要做其他的吗？我可以进一步：生成这篇论文的方法思维导图、与某篇具体论文做对比分析、帮你整理 related work 脉络、或者起草 paper note。"
